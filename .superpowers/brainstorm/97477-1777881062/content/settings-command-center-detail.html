Design proposal

Settings becomes canonical: the UI edits one merged config object, validates it before save, and saves with conflict metadata. The optimiser, reporter, and rules views read from that config instead of treating generated defaults as equally authoritative.

Bulk operators are contextual: each matrix supports row, column, and selected-cell operations. Trainer tables get column toggles and filtered bulk edits. Every operation shows a preview before mutation.
